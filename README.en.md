# carton

`carton` is a project introduction built around the packaging logic of cartons.

In this description, a carton is not just a container. It is a packaging system and a fast packing and assembly method.

Its value is centered on three actions:

- Quickly gather loose contents
- Quickly form an outer structure
- Quickly move into a state that is ready for transport, stacking, and delivery

From this perspective, a carton is more than packaging material. It is a lightweight assembly solution for real-world handling problems. It turns exposed, scattered, hard-to-move items into a single unit that can be transported, displayed, stored, and reopened with ease.

## System Overview

`carton` is a carton system designed for packaging, packing, and assembly scenarios. It focuses on using direct, stable, and low-cost methods to turn separate items into an organized whole. Its core concerns include:

- What kind of structure allows packing to be completed in the shortest time
- What folding method is best for fast assembly
- What dimensions work best for carrying, stacking, and repeated use
- What appearance most clearly communicates that the contents are already organized

## Design Principles

- Fast: prioritize packing and shaping first
- Lightweight: solve problems with the simplest possible structure
- Easy to assemble: avoid unnecessary steps
- Enclosable: quickly gather scattered items into one unit
- Reopenable: easy to ship when closed and easy to use when opened
